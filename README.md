# money-back-my-home

**蒙地卡羅美股策略分析 v2**

輸入美股代碼與目標利潤,工具比較「六種進場點位 × 四種出場規則」共 24 種組合,
以**機會成本入帳的帳戶年化**與「單純持有」正面對決,並用三層驗證(全期實測、
walk-forward 樣本外、熊市壓力測試)告訴你結果可不可信 — 包括誠實告訴你
「這檔股票其實不該擇時」。

## 啟動

```
pip install -r requirements.txt
python app.py          # 或直接雙擊 start.bat
```

瀏覽器開 http://127.0.0.1:5000

網址直接帶參數:`/?ticker=NVDA&target=20`;多檔掃描:`/?ticker=AAPL,NVDA,MSFT&target=15`

## 功能

- **六種進場**:現價直接進場,或等回檔 3% / 5% / 8% / 10% / 15% 掛限價單
- **四種出場**(一次全算,前端即時切換):
  - 固定目標:達標即以目標價賣出
  - 固定目標 + 停損
  - 移動停利:達標後不賣,自高點回落 N% 才出場(讓利潤奔跑)
  - 移動停利 + 停損(預設建議)
- **帳戶年化**:同一資金窗口(等待 + 最長持有),未觸發與出場後的閒置資金以現金利率
  計息,全部路徑取幾何平均後年化 — 等回檔的機會成本與提前出場的再投資斷檔因此入帳,
  和「單純持有」基準直接可比;輸給基準的組合會標灰並顯示警示
- **¼ Kelly 建議倉位**:由模擬的單次交易報酬分佈推得
- **三層驗證**:
  1. 全期實測(in-sample):同規則在全部歷史逐段執行,驗證模擬器一致性
  2. walk-forward(樣本外):只用前半資料建模 → 後半資料實測,估計真實預測力
  3. 熊市壓力測試:只從歷史最差一年抽樣重新模擬,看停損的保險價值
- **多檔掃描**:逗號分隔多個代碼,逐檔找出最佳組合並依帳戶年化排序 —
  把資金配到當下勝率 × 賠率最好的標的(廣度才是提高組合年化的正道)
- **市場探索(新手模式)**:一鍵掃描內建約 110 檔美股大型股與 ETF(`universe.py`,可自行增刪),
  自動分成三類並給白話行動卡:
  1. 擇時有優勢 — 策略帳戶年化「同時」贏過單純持有與現金 +2pp,入圍者自動跑完整
     walk-forward 與壓力測試,給高/中/低信心分級
  2. 適合買入持有 — 擇時只會損耗,直接買著別動(附 ¼ Kelly 倉位)
  3. 建議避開 — 持有與擇時都不如現金

## v3 新增(準確率與誠實度)

- **條件化模擬**:依當前市場狀態(現價 vs 200 日均線 × 近 21 日波動 vs 歷史中位)
  只從歷史上相似狀態的時期抽樣;條件僅作用於前一季(約 63 個交易日),之後回歸
  無條件 — 狀態資訊的可預測性會衰減,鎖死整個窗口反而高估。樣本外驗證採
  「逐起點查表」:後半段每個起點依當時狀態取對應預測,驗證的是實際使用情境。
- **68% 信賴區間**:二階 bootstrap(重抽替代歷史 × 重跑模擬)估計帳戶年化的模糊度,
  顯示在建議卡與出場規則對比表 — 十年歷史的點估計天生就有 ±5~12pp 的寬度,
  區間外仍有約 1/3 機率。
- **建議追蹤(成績單)**:按「追蹤」記錄任何建議(SQLite 持久化),之後自動以
  收盤資料推進狀態機(等待觸發 → 持有 → 出場)並對答案;`/?track=1` 直接開啟。
- **組合層相關性**:掃描結果自動偵測近一年報酬相關 > 0.8 的群組(同產業漲跌同步),
  提醒以「群」為單位分配倉位。
- **組合輪動回測**:在掃描標的的真實十年歷史上模擬「出場後資金立即輪動到下一個
  訊號」,解除單檔帳戶年化「出場後閒置」的保守假設 — 這才是把工具當完整策略用
  時實際能拿到的年化(標的清單由同段歷史選出,結果偏樂觀,參考結構而非絕對值)。
- **小項**:厚尾增強(bootstrap 疊加 t(4) 噪音,可產生略超歷史極值的情境)、
  股息預扣稅(預設 30%,以近一年實際配息計算拖累)、資料健檢(過期/極端跳動/缺口警告)。

## 方法論

蒙地卡羅採 block bootstrap(區塊 21 個交易日,保留波動聚集),自現價模擬數千條路徑,
在每條路徑執行完整進出場規則。限價觸發、停損、移動停利皆以收盤價判定(偏保守);
達標以目標價成交、停損/移動停利以當日收盤成交;未含交易成本(股息稅可設)。

已知限制(誠實聲明):bootstrap 假設未來重演過去的報酬分佈,對倖存者偏差無解
(你只會輸入現在還活著的股票);回測樣本重疊,有效獨立樣本遠少於表面數量;
walk-forward 偏差通常比 in-sample 大,那才是接近真實的預測誤差;
條件化與厚尾增強能縮小但無法消除這些誤差。

## 檔案

| 檔案 | 說明 |
|---|---|
| `engine.py` | 模擬 + 四種出場 + 條件化 + 區間 + 三層驗證引擎(CLI:`python engine.py NVDA 20`) |
| `rotation.py` | 組合輪動回測(事件驅動,多檔資金再部署) |
| `tracker.py` | 建議追蹤與自動對答案(SQLite:`tracker.db`) |
| `app.py` | Flask 伺服器與 API(`/api/analyze`、`/api/discover/*`、`/api/rotation`、`/api/track/*`) |
| `static/index.html` | 前端儀表板(Chart.js) |

## 免責聲明

本工具為歷史資料統計模擬,所有數字皆為估計,不代表未來表現,不構成投資建議。
Kelly 倉位為理論上限的四分之一,仍可能高於你的風險承受度。資料來源:Yahoo Finance。
